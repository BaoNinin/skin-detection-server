# 历史记录功能测试指南

## 问题排查步骤

### 1. 确认后端接口正常

```bash
# 测试后端接口
curl "http://localhost:3000/api/skin/history?userId=14"
```

**预期结果**：
```json
{
  "code": 200,
  "msg": "查询成功",
  "data": [
    {
      "id": 5,
      "skin_type": "敏感性皮肤",
      "concerns": ["泛红", "刺痛"],
      "moisture": 50,
      "oiliness": 35,
      "sensitivity": 70,
      "created_at": "2026-03-15T11:20:00.000Z"
    },
    // ... 更多记录
  ]
}
```

### 2. 确认用户已登录

在小程序开发工具中：

1. 打开"我的"页面（`/pages/profile/index`）
2. 点击"登录"按钮
3. 查看控制台日志，确认用户信息已保存

**预期日志**：
```
登录成功
用户信息已保存: { id: 14, ... }
```

### 3. 确认 userId 已保存

在小程序开发工具的"Storage"标签页中：

1. 找到 `userId` 键
2. 确认值存在（例如：`14`）

或者使用控制台：
```javascript
Taro.getStorageSync('userId')
// 应该返回数字 14 或字符串 '14'
```

### 4. 查看历史记录页面

1. 点击底部"历史"标签（`/pages/history/index`）
2. 查看控制台日志

**预期日志**：
```
历史记录页面 - userId: 14 类型: string 是否有效: true
历史记录页面 - 请求 URL: /api/skin/history?userId=14
历史记录页面 - 响应数据: { code: 200, msg: "查询成功", data: [...] }
历史记录页面 - 查询成功，记录数: 6
```

### 5. 检查页面显示

**时间轴视图**：
- 显示 6 条检测记录
- 每条记录显示：皮肤类型、检测时间、各项指标
- 可以点击查看详情或选择对比

**日历视图**：
- 显示当前月份日历
- 有检测记录的日期显示红色圆点
- 点击日期可以查看当天的检测记录

## 常见问题

### Q1: 页面显示"暂无检测记录"

**可能原因**：
1. 用户未登录（userId 为空）
2. 用户 ID 与测试数据不匹配

**解决方法**：
1. 先登录：点击"我的" → "登录"
2. 确认 userId 正确（应为 14）
3. 查看控制台日志确认请求参数

### Q2: 网络请求失败

**可能原因**：
1. 后端服务未启动
2. 网络配置问题

**解决方法**：
1. 检查后端服务是否启动（localhost:3000）
2. 检查环境变量配置（PROJECT_DOMAIN）
3. 查看控制台网络请求日志

### Q3: 数据显示不完整

**可能原因**：
1. 数据格式问题
2. 前端解析错误

**解决方法**：
1. 查看后端返回的完整数据
2. 检查控制台是否有错误
3. 重新编译小程序

在微信开发者工具中：
1. 打开"Console"标签页
2. 筛选关键词：`历史记录`、`Network`

### 查看后端日志

```bash
tail -f /tmp/coze-logs/dev.log | grep "历史记录"
```

### 直接测试后端接口

```bash

# 保存历史记录

    "concerns": ["干燥"],

    "oiliness": 30,
    "sensitivity": 40,
    "recommendations": ["加强保湿"]
  }'
```
