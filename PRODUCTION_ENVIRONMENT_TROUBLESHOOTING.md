# 🔥 小程序上线后功能失效问题排查

## 问题现象

- ✅ 体验版：功能正常
- ✅ 开发版：功能正常
- ❌ 正式版（上线后）：功能失效

---

## 🎯 最可能的原因：服务器域名未配置 ⭐⭐⭐

### 原因分析

**开发版和体验版为什么能用？**

在微信开发者工具中，可以开启「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」：
- 开发版：默认不校验
- 体验版：可以配置不校验
- **正式版：强制校验** ⚠️

**正式版要求：**
- 必须在微信小程序后台配置服务器域名白名单
- 所有网络请求必须使用 HTTPS
- 域名必须在合法域名列表中

---

## 🔍 快速诊断

### 症状 1：网络请求失败

**表现：**
- 小程序加载正常
- 点击功能按钮无响应
- 控制台报错：`request:fail` 或 `url not in domain list`

**原因：** 服务器域名未配置

---

### 症状 2：API 返回错误

**表现：**
- 网络请求可以发送
- 但返回错误数据或 404

**原因：** 域名配置错误或服务未正确部署

## 🔧 其他可能原因

### 原因 1：环境变量配置问题

**检查 .env.local 配置：**

```bash
# 检查 PROJECT_DOMAIN 是否正确
PROJECT_DOMAIN=https://your-domain.com
```

**检查 config/index.ts：**

```typescript
defineConstants: {
  PROJECT_DOMAIN: JSON.stringify('https://your-domain.com'),
  TARO_ENV: JSON.stringify(process.env.TARO_ENV),
  TARO_APP_WEAPP_APPID: JSON.stringify('wx8826c7b681ec3c65'),
}
```

**确保：**
- 正式版和开发版使用相同的域名
- 域名可以正常访问
- HTTPS 证书有效

---

### 原因 2：云托管服务配置问题

**检查云托管服务状态：**

1. 登录云托管控制台
2. 检查服务 `skin-detection-server` 是否运行正常
3. 检查服务实例是否健康
4. 检查服务日志是否有错误

---

### 原因 3：HTTPS 证书问题

**检查 HTTPS 证书：**

1. 在浏览器中访问你的域名
2. 检查是否有证书错误
3. 检查证书是否过期
4. 检查证书是否信任

**测试 HTTPS：**
```bash
curl https://your-domain.com/api/health
```

1. 在正式版小程序中打开 vConsole
